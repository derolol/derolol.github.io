无状态地址自动配置(SLAAC)

  • 基于 NDP (Neighbor Discovery Protocol)

  • 无需任何配置即可和外界通信，即插即用，没有维护网络地址信息的服务器

  • 地址结构

    • 前缀
      • 一般为路由器的前缀
    • 接口 ID
      • EUI-64
      • 随机
      • 静态

  • 获取配置信息

    • Router Solicitation (RS)

      由主机通过组播的方式发出消息，促使路由器发送 RA 消息

    • Router Advertisement (RA)

      向主机告知路由器前缀

两种信息 ICMP 报文的形式出现

  • 配置中的机制

    • 路由器发现

      • 主机选择默认网关
      • 主机发现前缀，生成前缀列表
      • 参数发现：发现相关参数的过程，如 MTU、跳数限制、地址配置方式等

    • 重复地址检测(DAD)

    • 前缀重新编址

路由器发现实例

  • 主机请求——触发路由器

注意

  1. 避免 RS 泛滥，节点启动时最多智能发送 3 个 RS

  2. 主机收到 RA 时，应该设置默认路由器，建立默认路由器列表、前缀列表及其他参数

  3. 主要周期性发送 RA （默认值 200s）

  4. IPv6 地址在系统中有一个生存周期，跟优先时间和有效时间有关

    链路本地地址的优先时间和有效时间是无限的，永不超时

  5. RA 的前缀选项指示了生存期

    • 在 preferred lifetime 不受限制使用
    • 超过 preferred lifetime，但仍处于 valid lifetime，新上层应用不可用
    • 超过 valid lifetime，任何都不可用

  6. RA 消息的配置选项

    • M Managed Address Configuration 管理地址配置标识
    • O Other stateful Configuration 默认配置
    • A 用于指示终端是否能使用该前缀进行无状态地址自动配置

DAD(Duplicate Address Detection)

  • 背景

    • 地址可能不唯一

  • 概述

    • 重复地址检测
    • 是节点确定即将使用的地址是否在链路上唯一的过程
    • 所有的 IPv6 单播地址，都必须要通过 DAD
    • DAD 机制通过 NDP 的 NS、NA 两种消息实现
    • 一个地址在非配给一个接口之后，通过 DAD 之前成为 tentative 地址
    • 处于 ALL-NODES 组和 Solicited-Node 组播组中

  • 基本思想

    1. 节点组播发送 Neighbor Solication
    2. 如果收到 Neighbor Advertisement，就证明地址重复
    3. 若尝试若干次发送请求未收到通告，则无重复

DAD 实例

注意

Neighbor Solication 接收者如果发现 Target 域中的地址对它而言是 tentative，则主动放弃使用

前缀重新编址

  • 允许网络从以前的前缀平稳过渡到新前缀，对用户透明
  • 路由器通告 RA 当前前缀和新前缀，当前前缀的优先时间和有效时间被减小到接近 0，开始通告新的前缀
  • 节点收到 RA，发现生命周期较短，停止使用，使用新的前缀配置接口，DAD 后获取新地址

注意

  • 转换期间，节点有两个单播地址使用

    • 旧地址：基于旧的前缀，用于维持以前已经建立的连接
    • 新地址：基于新的前缀，用来建立新的连接

  • 当旧的前缀的有效时间递减为 0 时，旧的前缀完全废止

有状态地址自动配置(DHCPv6)

背景

  • 更好地管理地址

一般特点

  • 使用 UDP 交换报文，端口 546/547
  • 使用链路本地地址或其他机制获得的地址来发送和接收 DHCPv6 报文
  • 客户机只需发送给保留的链路范围组播地址
  • 取消 DHCPv4 中的 Discover 和 Offer 信息

DHCPv6 的快速地址配置

  • 只需要两个消息的交互

  • 异常情况处理

    • 客户端发送 Confirm 确认是否可继续使用

DHCPv6 的重配置机制

DHCPv6 报文

  • msg-type 报文类型
  • transaction
  • options

自动地址配置

  • SLAAC
  • AMO = 101
  • M = 1
  • 对比
方式 地址管理 部署价值 实现难度 安全性
DHCPv6 有状态，服务器存储用户地址或前缀的分配信息 支持 128bit 地址和不同长度的前缀分配，扩展性强 配置复杂 应用层协议，安全性强
SLAAC 无状态，路由器不保存用户地址分配信息 只支持 64bit 前缀配置，扩展性弱 配置简单 安全性差

IPv6 地址子网规划

  • IPv4 的子网划分是管理地址稀缺性
  • IPv6 子网划分是分局路由器的数量及他们所支持的网络来构建寻址分层结构

IPv6 分组格式

一般格式

  • 分组构成

    • 基本头部

      • Next Header 指向下一个扩展头或数据段

    • 扩展头部

      • 扩展头之间存在先后顺序关系

      • 是可变长的

      • 类别

        • 逐跳选项 HBH

    • 有效载荷

IPv6 +

概述

  • 基于 IPv6 下一代互联网的全面升级

    • 万物互联到万物智联
    • 由消费互联网向产业互联网升级

  • 内容

    • 协议创新

      SRv6、网络切片、随流检测、新型组播等

    • 网络智能化技术创新

  • 拓展内容

    • 郭贺铨院士谈以 IPv6+赋能智能云网

      • 高带宽
      • 确定性
      • 低时延
      • 安全

    • 华为

内容介绍

  • 网络切片

    • 在一张物理网络上提供多张逻辑网络（切片）

      满足网络隔离需求并提供服务质量保证

    • 每个网络切片可自定义逻辑拓扑、SLA 需求、可靠性安全和安全等级

  • 新型组播

    • BIERv6
    • 采用了目的选项扩展头 DOH 来携带组播信息

最大优势

  • 完全基于原始的 IPv6，无需单独部署网络，几乎利用 IPv6 的扩展头完成
  • 由中国标准作为技术创新交流与标准制定的基础平台

  • 主要用于网络检测

与 ICMPv4 的对比

    • 邻居发现
    • 无状态地址配置
    • 路径 MTU 发现（PMTU）
