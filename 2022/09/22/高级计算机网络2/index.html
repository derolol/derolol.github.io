高级计算机网络2

IPv6 协议

背景

IPv4 存在较多的不足

  • 地址数量不充足及地址使用不充分的问题
  • 端到端业务模式无法实施
  • 路由表的膨胀
  • 移动性支持不够，不同位置需要切换不同的地址
  • QoS 和性能问题
  • 配置复杂
  • 安全问题

主张替换 IPv4 而不是修补

IPv6 的诞生

  • 1991 RFC1287 未来的 Internet 体系结构
  • Steve Deering 在 IPv4 的基础上提出了 SIPP（简单增强 IP），可称为 IPv6 的前身

其他相关名词

  • Internet 2 下一代互联网
  • NGN/NGI 针对电信/计算机网络
  • The Second Internet 侧重于物联网
  • IPv1/IPv2/IPv3/IPv5 存在但无推广
  • IPv9 十进制网络

    • 通路 -> 通车，能用 -> 好用

IPv6 概述

IPv6 的特点

  • 地址及寻址
  • 全新的报文格式，高效的报头
    • 路由器的处理更高效
  • 全新的地址配置方式，即插即用
  • 更好的 QoS 支持
  • 内置的安全性
  • 全新的邻居发现协议
  • 良好的扩展性
  • 内置的移动性
  • 端点分片

TCP/IPv6 参考模型

  • 应用层 HTTP FTP DHCPv6
  • 传输层 TCPv6 UTPv6
  • 网络层 IPv6 ICMPv6 IPsec
  • 连接层

IPv6 基本术语

  • 链路: 到达内部子网路由器的端口前
  • 子网: 通过路由器与外部链接

IPv6 地址

  • 128 位地址

IPv6 地址数量

$$
\begin{split}
2^{128} &= 3.4 * 10^{38} \
&=340 * 10^{36}
\end{split}
$$

340 涧（1 涧 = $10^{36}$）

IPv6 地址格式

  • 冒分十六进制

    2001:0db8:85a3:08d3:1319:8a2e:0370:7344

  • 地址压缩

    • 省略前导 0
      • 2001:0db8:02de:0000:0000:0000:0000:0e13
      • 2001:db8:2de:0:0:0:0:e13
    • 忽略全 0
      • 2001:db8:2de:0:0:0:0:e13
      • 2001:db8:2de::e13

IPv6 地址分类

IPv6 地址分类 —— 单播地址

  • 用于唯一标识支持 IPv6 的设备上的接口

  • 源 IPv6 地址必须是单播地址

  • 单播地址分类

    • 链路本地地址
    • 环回地址
    • 未指定地址
    • 唯一本地地址
    • 嵌入式 IPv4 地址
    • 可聚合全球单播地址

链路本地地址

1. 概述

  • 链路本地地址用于与同一链路中的其他设备通信

  • 路由器不会转发具有本地链路源地址或目的地址

  • 每个支持 IPv6 的网络接口一定有链路本地地址

  • IPv4 链路本地地址不会自动生成，而支持 IPv6 的主机会自动创建链路本地地址

    这允许支持 IPv6 的设备与同一子网中的其他支持 IPv6 的设备通信，包括与默认网关（路由器）通信

  • 使用范围

    只能在同一本地链路节点之间使用

2. 构成

  • 前 64 位 FE80::/64

  • 后 64 位 EUI-64 地址

    直接指定或由 MAC 地址派生而来

3. 应用

  • 主机使用本地路由器的链路本地地址作为默认网关 IPv6 地址

  • 路由器使用链路本地地址交换动态路由协议信息

  • 转发 IPv6 数据包时，路由器的路由表使用链路本地地址确定下一跳路由器

  • 查看本地链接 IPv6 地址 ipconfig

    符号 % 表示网络接口号 interface

环回地址

  • 压缩地址表示 ::1/128 或 ::1
  • 主机使用环回地址发送数据包到自身
  • 不能分配给物理接口
  • 通过ping命令测试本地主机的 TCP/IPv6 配置

未指定地址

  • 压缩地址表示 ::/128 或 ::
  • 若设备尚无永久 IPv6 地址时，未指定地址可用作源地址，不能用作目的地址
  • 不能分配给接口

唯一本地地址

1. 概述

  • 唯一本地地址在一个站点内或者有限站点数之间用作本地地址
  • 在全局 IPv6 中不具有可路由性

2. 构成

  • 地址范围 FC00::/7 到 FDFF::/7
  • L: 标志位，1 表示地址在本地网络范围内使用，值 0 用于以后扩展
  • Global ID: 全球唯一前缀，通过伪随机方式产生
  • Subnet ID: 子网 ID，划分子网使用
  • Interface ID: 接口标识

嵌入式IPv4地址

  • 从 IPv4 过渡到 IPv6 时使用

可聚合全球单播地址

字段名 长度 描述
Prefix 48 Global Routing Prefix，ISP 商分配的前缀，含网络号用于路由，前三位为 001 表示为单播地址
Subnet ID 16 标识站点内部的子网
Interface ID 64 一个接口的唯一标识

  • 全球范围内实现单播，不是必选项

  • 接口 ID 的生成

    • EUI-64
    • 随机
    • 手工

IPv6 地址分类 —— 组播地址

  • 组播地址结构

    • Flags 0001 或 0000
    • Scope 组播范围
    • Group ID 组播 ID

  • 常见组播地址

IPv6 IPv4 组播组
FF02::1 224.0.0.1 链路上的所有节点
FF02::2 224.0.0.2 链路上所有路由器地址

  • 特殊的组播地址

    Solicited-node 节点地址：被请求节点组播地址，主要用于重复地址检测和获取邻居节点的链路层地址

    • 地址构成

      • 前 104
      • 后

凡是单播地址后 24 位相同的接口自动加入相应的请求节点组播组

IPv6 地址分类 —— 任播地址

  • 目标地址为仁波地址的数据包将发送给最近的一个接口
  • 适合于 One-to-One of Many
  • 任播地址从单播地址空间中分配，用于标识一组网络接口
  • 只能作为目标地址，且进分配给路由器

一个接口上的地址：多地址

一台 IPv6 主机，其一个接口上可以具有的 IPv6 地址有
链路本地地址
环回地址
所有节点组播地址
分配的可聚合全球单播地址
被请求节点组播地址
主机所属组的组播地址
